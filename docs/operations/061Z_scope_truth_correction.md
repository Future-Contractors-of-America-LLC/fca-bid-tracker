# 061Z scope truth correction

Status: active  
Date: 2026-06-15

## Issue

Deployment-closeout work was allowed to sit too close to product-completion language.
That created an unacceptable truth gap.

A successful deployment and proof bundle does **not** mean FCA already has:

- real lead/opportunity object interaction
- real qualification actioning
- real file/evidence handling
- real bid/estimate workflow entry
- real client portal task utility
- real customer-usable state mutation

## Truth

Current live state is deployment-valid but product-incomplete.
The shell, login, intake, and proof surfaces exist, but the customer-operable Contractor Command spine is not yet complete.

## Why this happened

Execution drifted into deployment continuity, proof persistence, workflow governance, and shell hardening while the flagship vertical slice remained only partially implemented.

The canonical build-sequence intent already required a phased product spine rather than an assumption of full completion:

- lock the shared spine first
- deliver one real vertical slice
- then deepen documents, takeoff, RFI/change, QC, finance, and Academy layers

## Non-negotiable correction

Do not describe FCA as complete SaaS + complete LMS until the following are true in live product truth:

1. lead/opportunity records can be created, qualified, updated, and persisted
2. bid/estimate records can be entered and advanced through real state
3. project/file/evidence objects can be attached, reviewed, and mutated
4. client portal tasks produce useful customer action
5. Academy has real enrollment/progress/completion utility tied to tenant/project context
6. customer login lands in a usable workflow, not commentary shell

## Immediate next build mode

Return to flagship product execution after deployment-closeout:

1. lead/opportunity + qualification vertical slice
2. bid/estimate + proposal state mutation vertical slice
3. file/evidence + audit-linked handling vertical slice
4. client task utility vertical slice
5. Academy/LMS operational depth vertical slice

## Reporting rule

From this point forward:

- deployment completion must be reported separately from product completion
- proof completion must be reported separately from customer utility
- live shell presence must not be treated as equivalent to real SaaS or real LMS
