# FCA Brand Identity Lock

## Decision
The FCA shell is now locked to the following brand model:

- **FCA / Future Contractors of America** = primary customer-facing platform brand
- **Auricrux** = embedded operating layer, intelligence surface, and continuity guide inside FCA

No customer-facing product surface should use any other brand.

## Visual direction
### FCA
- Primary color family: saturated FCA blue
- Role: trust, platform stability, operator confidence, production readiness
- Form language: geometric, structural, modular, contractor-system oriented
- Wordmark hierarchy: `FCA` first, `Future Contractors of America` supporting

### Auricrux
- Primary color family: gold / amber
- Role: guidance, intelligence, embedded system authority, next-action visibility
- Form language: emblematic, sovereign, active, protective
- Placement: secondary to FCA on customer-facing pages, primary only on Auricrux-specific surfaces

## Usage rules
1. FCA leads all public and customer-facing shell identity.
2. Auricrux appears as a system-within-the-system, not as a replacement for FCA.
3. FCA blue and Auricrux gold should remain visually distinct.
4. Tyler branding is forbidden on customer-facing routes, screens, copy, or assets.
5. Public shell, login, pricing, contact, platform, and portal entry should reinforce the same brand architecture.

## Implementation state
Locked into the public shell through:
- reusable FCA brand mark component
- reusable Auricrux brand mark component
- public page integration across home, platform, pricing, contact, login, header, footer, and the Auricrux route

## Follow-on replacement policy
If final exported logo files are added later, they should preserve this architecture:
- FCA remains the parent platform identity
- Auricrux remains the embedded operating layer identity
- color roles and naming hierarchy should stay consistent
