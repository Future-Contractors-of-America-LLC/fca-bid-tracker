# 062A Portal Alignment

## Portal truth already present
The portal contains real route-level slices for:
- qualification
- estimate progression
- proposal progression
- project progression
- file governance
- billing
- operations
- audit
- comms
- support
- admin
- Auricrux

## Portal alignment objective
The portal home and dashboard surfaces should present these as one contractor-command operating spine instead of as loosely related pages.

## Required portal grouping
- Sales / precon: bids, estimates, proposals
- Delivery / continuity: projects, files, audit, operations
- Commercial / customer: billing, messages, notifications, support
- Intelligence / system control: Auricrux, admin, profile
- Learning: academy

## Gate
Any portal navigation or summary card set that hides these real groups is below site-level alignment target.
