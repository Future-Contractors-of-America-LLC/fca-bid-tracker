## System Requirement

FCA must cover the complete construction lifecycle with no gaps.

External systems may be integrated for customer comfort, migration, or interoperability, but FCA must not depend on them for core system completeness.

Auricrux must be present across every module as the embedded execution, review, correction, and teaching intelligence.

## Core Spine Objects

Every applicable module must attach to:

- Tenant
- User / Role
- Project / Job
- Opportunity / Bid
- Files / Evidence
- Auricrux Action Log
- Financial Link
- Audit / Event Record

If a feature does not connect to the required spine objects where applicable, the feature remains incomplete.

---

## Core Rules

Every module must:

- link to Project / Job where applicable
- accept file input where applicable
- produce usable output
- log Auricrux actions
- remain reviewable and correctable
- expose truthful state, not simulated completion

---

## Auricrux Requirement

Auricrux must:

- read all relevant files, records, and workflow state
- detect gaps, failures, drift, and incompleteness
- recommend corrections
- execute lawful changes within scope
- record all material actions and outcomes

---

## Failure Condition

If any of the following is missing where applicable:

- project linkage
- file linkage
- output generation
- audit tracking
- truthful state labeling

then a system gap exists.

Auricrux must detect, report, and remediate the gap.
